# Product Positioning

## 一句话

`cc-gw` 帮你把 Claude Code、OpenAI SDK、Anthropic SDK 和内部工具统一到一个本地优先入口，让 Key、路由、日志和排查不再散落在每个项目里。

## 适合谁

- 个人开发者：不想在一堆脚本和工具里维护不同 provider 的 key、`baseURL` 和协议细节
- `2-10` 人 AI 小队：需要统一入口、日志排查、路由切换和基础权限控制，但不想先上重平台
- `10-100` 人软件研发团队：开始多人共享模型调用、做分环境接入和稳定性观察，已经需要一个轻量控制面

## 不适合谁

- 追求跨 BU 统一治理、复杂审批流、企业 SSO 和组织级策略中台的团队
- 希望把所有 AI 成本结算、身份体系和审批流程都放进一个大平台的组织

## 核心价值

- 一个统一入口承接 Claude Code、Anthropic / OpenAI SDK 和兼容 provider
- 模型路由、协议兼容和 provider 切换从业务代码里抽离
- Dashboard、Logs、Events、Profiler、API Keys 回到同一个控制台
- 本地优先、自托管友好，适合开发机、共享团队实例和轻量部署

## 推荐叙事

- 更像研发团队自己的日常 AI 工作台
- 不是平台委员会项目
- 先把入口、观测和路由收口，再逐步补治理

## 官网 / README / 发版文案应保持一致的点

- `GET /` 是产品官网和定位说明
- `GET /ui/` 是 Web 管理控制台
- 产品重点是个人开发者与小团队，不是大规模企业治理
- 主卖点是统一入口、少改业务代码、日志排查和轻量本地协作
